Lightning

JUnit report filename suffix

Note: This optional parameter is currently only supported by jmeter-lightning-maven-plugin.

The JUnit report filename suffix parameter allows you to specify a filename suffix for the JUnit XML report that Lightning produces. It is useful when multiple plugin executions are required, and more than one JUnit XML reports are to be produced, rather than a single junit.xml. Define the JUnit XML filename suffix with the <junitReportSuffix> tag. For example:

<plugin>
    <groupId>uk.co.automatictester</groupId>
    <artifactId>jmeter-lightning-maven-plugin</artifactId>
    <version>${jmeter-lightning-maven-plugin.version}</version>
    <executions>
        <execution>
            <id>test-a</id>
            <goals>
                <goal>lightning</goal>
            </goals>
            <configuration>
                <mode>verify</mode>
                <testSetXml>data/test-set-foo.xml</testSetXml>
                <jmeterCsv>data/results-foo.csv</jmeterCsv>
                <junitReportSuffix>foo</junitReportSuffix>
            </configuration>
        </execution>
        <execution>
            <id>test-b</id>
            <goals>
                <goal>lightning</goal>
            </goals>
            <configuration>
                <mode>verify</mode>
                <testSetXml>data/test-set-bar.xml</testSetXml>
                <jmeterCsv>data/results-bar.csv</jmeterCsv>
                <junitReportSuffix>bar</junitReportSuffix>
            </configuration>
        </execution>
    </executions>
</plugin>

In the configuration above, Lightning will be executed twice. The first execution will test results-foo.csv, producing a JUnit XML report named junit-foo.xml. The second execution will test results-bar.csv, producing a JUnit XML report named junit-bar.xml
